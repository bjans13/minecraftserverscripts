# Contributing to Minecraft Server Management Scripts

Thank you for your interest in contributing to this project! This document provides guidelines and information for contributors.

## How to Contribute

1. Fork the repository
2. Create a new branch for your feature or fix
3. Make your changes
4. Submit a pull request

## Guidelines

### Script Modifications
- Maintain the existing error handling structure
- Add appropriate comments for new functionality
- Test thoroughly before submitting
- Update documentation if adding new features

### Testing
- Test scripts in a safe environment first
- Verify backup and restore functionality
- Check for potential security implications
- Ensure proper error handling

### Pull Request Process
1. Update the README.md with details of changes if applicable
2. Update the version number in relevant files
3. Detail your changes in the pull request description
4. The repository owner will review and merge your changes

## Code Style
- Use clear, descriptive variable names
- Add comments for complex operations
- Follow existing formatting patterns
- Keep functions focused and modular

## Questions or Problems?
- Open an issue for bugs or suggestions
- Provide clear reproduction steps for bugs
- Include relevant system information
